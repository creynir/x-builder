# XOB-001: [CHORE] Workspace — add `playwright` runtime + `@x-builder/runner` + `@x-builder/overlay` packages

## Goal

Stand up the two new workspace packages the epic depends on — `@x-builder/runner`
(Node, drives Playwright + imports the engine in-process) and `@x-builder/overlay`
(bundled IIFE the runner injects) — and add the `playwright` runtime dependency.
This ticket is scaffolding only: `pnpm install`, `pnpm typecheck`, and `pnpm build`
must be green across the workspace, and the `x-builder` bin must resolve, with **no
runner or overlay feature logic present**.

Depends on: nothing. Unblocks XOB-002, XOB-015, XOB-018.

## Changes

Exact files / deps / configs:

- **`pnpm-workspace.yaml`** — add `"runner"` and `"overlay"` to the `packages:` list
  (alongside the existing `engine`, `client`, `shared`, `e2e-tests`, `tools/*`).
- **`runner/package.json`** (new) — `name: "@x-builder/runner"`, `private: true`,
  `"type": "module"`, `main`/`types` pointing at `./dist/...`, and a `"bin"` entry
  mapping `x-builder` → `bin/x-builder` (`{ "bin": { "x-builder": "./bin/x-builder.js" } }`).
  Scripts mirror `engine/package.json` exactly: `build` = `tsc -p tsconfig.json`,
  `typecheck`/`lint` = `tsc -p tsconfig.json --noEmit`, `test` = `vitest run`, `dev` = `tsx ...`.
  Dependencies: `"playwright": "^1.50"`, `"@x-builder/engine": "workspace:*"`,
  `"@x-builder/shared": "workspace:*"`. devDependencies mirror engine
  (`tsx`, `typescript ^5.7.0`, `vitest ^3.0.0`).
- **`runner/bin/x-builder`** (new) — the bin target. A **no-op stub is acceptable**
  for this ticket (e.g. prints a one-line "x-builder runner (stub)" and exits 0).
  Must be resolvable/executable so `npx x-builder` runs; must compile/typecheck.
  Use whatever entry shape (`.js` shim or compiled `.ts`) keeps `pnpm build`/`typecheck`
  green; the actual `RunnerApp.start()` wiring lands in XOB-015.
- **`runner/tsconfig.json`** (new) — extend `../tsconfig.base.json` exactly as
  `engine/tsconfig.json` does; emit to `dist/`.
- **`overlay/package.json`** (new) — `name: "@x-builder/overlay"`, `private: true`,
  `"type": "module"`. `build` produces a single bundled IIFE at `dist/overlay.iife.js`.
  Use **`vite`** in library/IIFE mode (`vite` is the established workspace bundler —
  `@x-builder/client` already uses `vite ^6.0.0`); `build` = `vite build`,
  `typecheck`/`lint` = `tsc -p tsconfig.json --noEmit`, `test` = `vitest run`.
  devDependencies: `vite ^6.0.0`, `typescript ^5.7.0`, `vitest ^3.0.0`.
  Dependency: `"@x-builder/shared": "workspace:*"` (overlay consumes the transport
  types only — no engine dep).
- **`overlay/vite.config.ts`** (new) — `build.lib` IIFE config with a single entry,
  output filename `overlay.iife.js`, `formats: ["iife"]`, a global name, into `dist/`.
- **`overlay/tsconfig.json`** + a **minimal placeholder entry module** (e.g. an empty
  `src/index.ts` exporting nothing meaningful) so `vite build` emits a non-empty
  `dist/overlay.iife.js`. No overlay UI/components — that is XOB-018+.
- **Turbo** — `turbo.json` `build` task already globs `dist/**`/`build/**` as outputs
  and runs `^build` first; both new packages inherit it via their `build` scripts.
  Add no per-package Turbo task overrides unless a package needs one to build green.
- **`pnpm-lock.yaml`** — regenerated by `pnpm install` (the new `playwright ^1.50`
  runtime entry must land here; `@playwright/test ^1.50.0` already exists under
  `e2e-tests` — the full `playwright` runtime lib is the genuinely new dependency).
- **`.gitignore`** — only if a new ignorable output path (e.g. `runner/dist`,
  `overlay/dist`) is not already covered by existing `dist/` globbing.

## Verification

Exact commands that prove success (run from repo root):

- `pnpm install` — resolves the workspace incl. `playwright ^1.50`; lockfile updates cleanly.
- `pnpm -F @x-builder/runner exec playwright --version` — prints a `1.50.x` version
  (proves the runtime lib is installed under the runner package).
- `pnpm typecheck` — green across all packages incl. `runner` + `overlay`.
- `pnpm build` — green across the workspace; `runner/dist/` and a non-empty
  `overlay/dist/overlay.iife.js` are produced.
- `npx x-builder` (or `pnpm -F @x-builder/runner exec x-builder`) — bin resolves and
  exits 0 (no-op stub OK).

## Integration Point

Reached by: the workspace tooling (`pnpm`/Turbo) and, later, the runner entry point.
`@x-builder/runner` is the package XOB-015 fills with `RunnerApp`/`BrowserController`
and that consumes `@x-builder/engine` in-process; `@x-builder/overlay`'s
`dist/overlay.iife.js` is the artifact XOB-015's `addInitScript` injects and that
XOB-018+ fill with the shadow-DOM React overlay. Terminal outcome of this ticket:
a clean checkout can `pnpm install && pnpm build` and resolve `x-builder` with both
packages present and empty.

## Scope Boundaries / Out of Scope

- **May change:** `pnpm-workspace.yaml`, `pnpm-lock.yaml`, `.gitignore`; create
  `runner/` and `overlay/` package skeletons (package.json, tsconfig, vite config,
  bin stub, placeholder entry); `turbo.json` only if a task override is strictly
  required to build green.
- **Excluded:** any change to `engine/`, `shared/`, `client/`, or `e2e-tests/`
  source. No bump of the existing `@playwright/test` version.
- **ZERO-TRACE (no code, no stubs beyond the no-op bin + empty entry):** no
  `RunnerApp`, `BrowserController`, `ExposeFunctionTransport`, `GraphQlCaptureObserver`,
  `XGraphQlNormalizer`, no `launchPersistentContext` call, no `addInitScript`, no
  `exposeFunction` binding, no overlay React/shadow-DOM/transport code, no
  `EngineTransport` import. The bin must not launch a browser or import the engine.

## Test Strategy & Fixture Ownership

- **Coverage level:** build/typecheck/bin-resolution smoke only — this is a `[CHORE]`;
  no unit suite is added.
- **Owning suite/workspace:** verification runs at the root via Turbo
  (`pnpm typecheck`, `pnpm build`) per existing convention; no new Vitest test files.
- **Fixture/helper strategy:** none — the placeholder overlay entry and no-op bin are
  the only new modules.
- **Dependency category:** `playwright` is **true-external** (a real browser-driver
  runtime), but at this ticket it is only *installed and version-probed*, never invoked
  to launch a browser — so no isolation boundary is exercised here.
- **Isolation boundary:** n/a — no runtime behavior to isolate; verification is the
  build graph itself.

## Definition of Done

- Both packages exist, install, typecheck, and build green across the workspace.
- `playwright ^1.50` is a `@x-builder/runner` runtime dependency and version-probes via the package.
- `x-builder` bin resolves and runs as a no-op.
- `overlay/dist/overlay.iife.js` is produced and non-empty.
- Zero runner/overlay feature logic present (zero-trace list above holds).

## Acceptance Criteria

- **Given** a clean checkout, **When** `pnpm install && pnpm build` runs, **Then** it
  completes green and both `@x-builder/runner` and `@x-builder/overlay` build, with a
  non-empty `overlay/dist/overlay.iife.js` emitted.
- **Given** the installed workspace, **When** `pnpm -F @x-builder/runner exec playwright --version`
  runs, **Then** it prints a `1.50.x` version (runtime lib present under the runner).
- **Given** the installed workspace, **When** `npx x-builder` runs, **Then** the bin
  resolves and exits 0 (no-op), launching no browser and importing no engine module.
- **Given** the workspace, **When** `pnpm typecheck` runs, **Then** it is green
  including the two new packages.
- **(Negative)** **Given** this ticket's diff, **When** the runner/overlay sources are
  inspected, **Then** none of `RunnerApp`, `ExposeFunctionTransport`,
  `GraphQlCaptureObserver`, `addInitScript`, `exposeFunction`, or any overlay React
  code is present (zero-trace).

## Edge Cases

- First `playwright` install may require browser binaries (`playwright install`) — **out
  of scope here**: this ticket only requires the JS lib + `--version`, NOT a downloaded
  Chromium. The first-run install flow is XOB-015's concern.
- IIFE bundle must be self-contained (no external imports at runtime) since it is
  injected via `addInitScript`; verify `vite` lib config externalizes nothing into the
  IIFE output.
- The `bin` shebang/exec bit must survive `pnpm install` so `npx x-builder` is runnable
  on macOS/Linux.
- Lockfile churn: ensure only the intended `playwright` runtime entry is added; do not
  inadvertently bump `@playwright/test`.
